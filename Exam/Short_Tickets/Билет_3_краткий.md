## Вопрос 1: Основная цель хемоинформатики. Классификация вычислительных методов по точности

> [!tip] Ключевые тезисы
> - Цель: данные → информация → знание для поиска и оптимизации **лидерных молекул**
> - По точности: квантовая химия > силовые поля > дескрипторные/ML методы
> - По типу: лиганд-базированные (быстрые, без структуры белка) vs структурно-базированные (медленные, точнее)
> - В реальном пайплайне методы комбинируются: быстрые фильтры → докинг избранных кандидатов

**Хемоинформатика** — это смешение информационных ресурсов для превращения химических данных в знание, нужное для **идентификации и оптимизации лидерных соединений** при разработке лекарств.

Представь шкалу «точность ↔ скорость»: чем точнее метод, тем он медленнее.

**Квантовая химия (DFT, ab initio)** — самая точная, считает электронные волновые функции. Но работает только с малыми молекулами (до ~50 атомов) и крайне медленна — для скрининга баз данных не годится.

**Силовые поля (молекулярная механика)** — атомы как шарики на пружинках. Быстрее в тысячи раз, используются в молекулярной динамике. Менее точны для химических реакций.

**Дескрипторные методы / ML** — молекула превращается в вектор чисел. Можно скринировать миллионы молекул за минуты (NeuroClick: **10 000–30 000 молекул/мин**). Основа **QSAR** и виртуального скрининга.

По типу: **лиганд-базированные** (нужны только активные молекулы, используют сходство и фармакофоры — быстро) vs **структурно-базированные** (нужна 3D-структура белка, **молекулярный докинг** — медленнее, но точнее).

## Вопрос 2: Метрики (Python), используемые в хемоинформатике

> [!important] Ключевые тезисы
> - Классификация: **ROC-AUC**, BEDROC, матрица ошибок → Precision/Recall/F1/MCC
> - Регрессия: **RMSE**, **R²**, **Q²** (кросс-валидированный R²)
> - Стандарт: разбивка **80/20**, **5-кратная кросс-валидация**
> - Библиотеки: `sklearn` (метрики + ML), `RDKit` (дескрипторы + фингерпринты), `PyRx` (докинг)

Модели в хемоинформатике решают две задачи: **классификацию** (активный/неактивный) и **регрессию** (предсказать число — IC₅₀, logBB, ΔGbind).

**Для классификации:** главная метрика — **ROC-AUC** (площадь под кривой чувствительность vs 1−специфичность). AUC = 1.0 — идеал, AUC = 0.5 — случайность. В статье Шитьякова: AUC = **0.82**. Лучшие ГЭБ-модели достигают AUC ≈ **0.959**. Из матрицы ошибок (TP/TN/FP/FN) считают Precision, Recall, F1 и **MCC** (хорош для несбалансированных классов). **BEDROC** — специальная метрика для виртуального скрининга: важно, чтобы активные молекулы оказались именно в топе списка, ведь тестируют только топ-N%.

**Для регрессии:** **RMSE** — средняя ошибка в единицах целевой переменной (чувствителен к выбросам). **R²** — доля объяснённой дисперсии (1 = идеал, <0 = хуже среднего). **Q²** — то же, но по кросс-валидации; если Q² << R², модель переобучена. Порог Q² > 0.5 считается приемлемым для QSAR.

**Валидация:** разбивка **80%/20%**, **5-кратная кросс-валидация** — стандарт. `sklearn.metrics` — для всех метрик, `RDKit` — для расчёта дескрипторов, `PyRx` — управление AutoDock Vina.

## Вопрос 3: Преимущества и недостатки молекулярных скаффолдов

> [!summary] Ключевые тезисы
> - **Скаффолд** = кольцевое ядро молекулы по Мурко (без заместителей)
> - Плюсы: организация БД, репозиционирование лекарств, комбинаторные библиотеки, снижение токсичности
> - Минусы: сужение химического пространства, синтетическая труднодоступность, наследование плохих ADME
> - **Scaffold hopping** — смена ядра при сохранении активности (обход патентов)

**Молекулярный скаффолд** — это ядро (core) структуры: кольцевые системы + линкеры между ними, без боковых заместителей. Определение **Мурко**, реализовано в RDKit: `GetScaffoldForMol()`. Заместители (R-группы) варьируют для подбора свойств.

**Зачем нужны скаффолды?**

*Организация баз данных (Scaffold Tree):* иерархия по ядрам упрощает субструктурный поиск и анализ SAR — видно, какие ядра порождают активные серии.

*Репозиционирование лекарств:* поиск скаффолда среди одобренных препаратов → быстрый выход на новые показания без синтеза с нуля (пример: скаффолд трамипросата при болезни Альцгеймера).

*Комбинаторные библиотеки:* одно ядро + список R-групп → сотни виртуальных производных. В click-химии (NeuroClick) продукты ещё и синтетически достижимы.

*Снижение токсичности (scaffold hopping):* замена ядра на изостерический аналог без **токсикофора** сохраняет активность и убирает нежелательный токсический потенциал.

**Ограничения:** фиксированное ядро сужает химическое пространство — первинные хемотипы так не найти. Генеративные модели часто предлагают несинтезируемые производные. Производные наследуют проблемы родителя: высокий LogP, плохую растворимость, реактивные метаболиты.

**Scaffold hopping** — переход к структурно новому ядру при сохранении биоактивности. Инструмент обхода патентной защиты и улучшения свойств.

## 📄 Статья: Shityakov & Förster (2014) — BBB Choline Transporter

> [!abstract] Карточка статьи
> **Цель:** создать in silico модель для предсказания транспорта катионных молекул через **BBB-ChT** (холиновый транспортёр ГЭБ) как способа доставки лекарств в ЦНС.
>
> **Метод:** BBB-ChT не кристаллизован → **гомологичное моделирование** по шаблону натрий/галактозного симпортера (PDB: **3DH4**). Докинг 93 соединений (44 активных + 49 неактивных) в **AutoDock** и **AutoDock Vina** через PyRx. Порог активности: **ΔGbind < −6.0 ккал/моль**.
>
> **Главный результат:** корреляция между расчётным и экспериментальным сродством **r² = 0.88**, AUC = **0.82** (AutoDock) / **0.81** (Vina), p < 0.0001, SEE = 0.775.
>
> **Вывод:** гомологичная модель + докинг — достаточная альтернатива QSAR для скрининга BBB-ChT-субстратов; активный транспорт через BBB-ChT значительно превосходит пассивную диффузию (logBBactive >> logBBpassive).

> [!note] Факты, которые точно спросят
> 1. **Size-related bias:** молекулы с **< 20 тяжёлых атомов** дают ложноотрицательный результат докинга — AutoDock плохо оценивает маленькие лиганды.
> 2. Дескрипторы модели: **ΔGbind, ClogP, PSA, logBBpassive, logBBactive** (с учётом экспериментального pKi).
> 3. Шаблон гомологичной модели — **не холиновый** белок, а **галактозный симпортер** (3DH4) — это частый вопрос-ловушка.
> 4. Датасет намеренно несбалансирован: 44 активных vs 49 неактивных — типичная ситуация реального скрининга.
