## Вопрос 1: Алгоритмы молекулярного докинга

> [!tip] Ключевые тезисы
> - Три класса: **стохастические** (GA, MC, SA), **систематические** (Glide, DOCK), **ML** (DiffDock, GNINA)
> - **Генетический алгоритм** — основа AutoDock/Vina: кроссовер + мутация + отбор по энергии
> - **Монте-Карло**: случайные ходы + критерий Метрополиса → умеет выходить из локальных минимумов
> - **DiffDock** — лучший результат: **38% top-1** (RMSD < 2 Å), но без физической скоринговой функции
> - Все классические методы используют **скоринговую функцию** для оценки позы лиганда

Молекулярный докинг — это предсказание того, как лиганд (малая молекула) «садится» в активный центр белка. Алгоритм должен решить две задачи: **перебрать возможные позы** лиганда и **оценить**, какая из них лучшая.

**Генетический алгоритм** (AutoDock, Vina) работает как эволюция: лиганд кодируется как «хромосома» (позиция + торсионные углы), популяция поз эволюционирует через кроссовер, мутацию и отбор по энергии ΔG. **Монте-Карло** делает случайные шаги и принимает «невыгодные» ходы с вероятностью exp(−ΔE/kT) — это позволяет не застрять в локальном минимуме. **Simulated Annealing** — похожая идея, но «температура» постепенно снижается: сначала система активно исследует пространство, потом «замерзает» около минимума.

**DiffDock** (MIT) — диффузионная модель: из случайной позы через обратную диффузию находит правильную. **38% успеха** против 23% у классики. **GNINA** — гибрид: MC-поиск + CNN-скоринг, +5% точности vs AutoDock. **EquiBind/TankBind** — регрессионные DL-модели, быстры, но не точнее классики.

## Вопрос 2: Принципы ML в молекулярном докинге

> [!tip] Ключевые тезисы
> - ML-модели обучаются на **PDB/PDBBind** (~20 000 экспериментальных структур)
> - **Регрессионные** (EquiBind, TankBind): предсказывают координаты напрямую — быстро, но неточно
> - **Генеративные** (DiffDock): моделируют *распределение* поз, а не одну точку — точнее
> - **GNINA**: традиционный поиск + **CNN-scoring** вместо эмпирической функции
> - Главная проблема всех ML-методов — **нет надёжной физической скоринговой функции**

ML вошёл в докинг в двух ролях: замена скоринговой функции и предсказание позы «с нуля».

**Регрессионные модели** (EquiBind на graph neural networks, TankBind) учатся минимизировать разницу между предсказанными и реальными координатами лиганда. Работают за миллисекунды против минут у AutoDock, но дают стерические клэши и не превосходят классику по точности (~20% vs 23%).

**DiffDock** — генеративная модель в пространстве трансляций (R³), ротаций (SO(3)) и торсий (Tⁿ). Forward pass «зашумляет» правильную позу, reverse pass убирает шум. Отдельная **confidence model** ранжирует результаты. Итог: **38% top-1**. Но рецептор обрабатывается как карта, а не атомная структура → стерические клэши, аффинность не предсказывается.

**GNINA** — форк AutoDock smina: MC генерирует позы, **CNN rescoring** заменяет эмпирическую функцию. Улучшение ~5%. Перспектива: объединить DiffDock (качество поз) + GNINA (физический scoring).

| Метод | Точность (RMSD<2Å) |
|---|---|
| AutoDock / Vina | ~23% |
| EquiBind / TankBind | ~20% |
| GNINA | ~28% |
| **DiffDock** | **~38%** |

## Вопрос 3: Недостатки генетических алгоритмов в докинге

> [!tip] Ключевые тезисы
> - **Жёсткий рецептор**: белок не меняет конформацию → нет induced-fit
> - **Ограниченный grid box**: blind docking практически невозможен
> - **Size-related bias**: малые молекулы занижены, крупные — завышены
> - **Экспоненциальный рост** пространства с числом торсионных связей
> - **Преждевременная сходимость** в локальном минимуме при малом числе итераций

> [!example] Аналогия
> ГА — как поиск ключа в комнате с закрытыми глазами: эффективен, если комната маленькая (жёсткий лиганд, известный сайт), но теряется в большом доме (много торсий, слепой докинг).

Генетический алгоритм — мощный и хорошо протестированный инструмент, но у него есть системные ограничения.

**Жёсткий рецептор** — самый принципиальный недостаток: белок в ГА зафиксирован, алгоритм не учитывает **induced-fit** (подстройку белка под лиганд). Если активный сайт меняется при связывании — результат будет неверным.

**Ограниченный поисковый объём**: AutoDock требует задать grid box (до 60 Å). При blind docking пространство огромно, алгоритм неэффективен. Неправильно выбранный центр сетки = ненайденный сайт.

**Size-related bias**: скоринговая функция некорректно ранжирует по размеру — малые молекулы (< 20 атомов) «недооцениваются», крупные «переоцениваются» из-за большего числа контактов с рецептором.

**Экспоненциальное пространство**: каждая вращаемая связь умножает число конформаций. AutoDock ограничивает торсии (~32 по умолчанию). Для больших лигандов глобальный минимум не гарантирован.

**Преждевременная сходимость**: при малом числе GA-runs алгоритм застревает в локальном минимуме. Для сложных систем (напр. циклодекстрин) нужно увеличивать число итераций до 100.

**Скорость**: 1–5 мин/лиганд на CPU — неприемлемо для скрининга библиотек в миллионы молекул.

## 📄 Статья: Shityakov et al. 2021 — Scaffold Searching in Alzheimer's Disease

> [!abstract] Карточка статьи
> **Цель:** Найти среди одобренных FDA/EMA препаратов кандидаты для репозиционирования как ингибиторы Aβ(1–42) при болезни Альцгеймера
>
> **Метод:** Скаффолд-поиск по структуре **трамипросата** (TRA) в базе **DrugCentral 2021** (4 642 препарата) → докинг AutoDock 4.2.5.1 → rescoring Glide/MM-GBSA → 100 нс молекулярная динамика (AMBER 16)
>
> **Главный результат:** Из 13 хитов два лидера — **менадион бисульфит** (ΔG = **−7.11 ккал/моль**) и **камфотамид** (ΔG = **−6.82 ккал/моль**) — превзошли референс TRA (ΔG = **−6.19 ккал/моль**) и по аффинности, и по проницаемости ГЭБ
>
> **Вывод:** Предложен валидный *in silico* пайплайн для раннего drug repurposing при болезни Альцгеймера

> [!note] Факты, которые точно спросят
> - **Мишень:** Aβ фибриллы 1–42, PDB: **2NAO** (цепи A, B, C)
> - **Референс:** Трамипросат (TRA) — Aβ-ингибитор, **провалился в фазе III** клинических испытаний
> - **База:** DrugCentral 2021, n = **4 642** препарата
> - **MM-GBSA**: менадион ΔG_GB = **−19.71** vs TRA **−10.1 ккал/моль** — почти двукратное превосходство
> - **logBB**: оба хита лучше TRA по проницаемости гематоэнцефалического барьера
> - Пайплайн: скаффолд-поиск (MolsoftICM) → AutoDock → Glide → AMBER MD
