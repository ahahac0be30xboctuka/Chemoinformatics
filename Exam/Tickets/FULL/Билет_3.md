## ❓ Вопрос 1: Основная цель хемоинформатики. Классификация вычислительных методов по точности.

> [!tip] Ключевые тезисы
> - Цель: трансформация данных → информация → знание для быстрого принятия решений при идентификации лидерных молекул
> - Классификация: по точности (квантовая > силовые поля > дескрипторные) и по типу (лиганд-базированные vs структурно-базированные)
> - Быстрые методы работают с лигандами; медленные — с комплексами «белок–лиганд»

### Развёрнутый ответ

**Основная цель хемоинформатики** — смешение разнородных информационных ресурсов для трансформации «сырых» химических данных в информацию, а информации — в знание. Это знание направлено прежде всего на **идентификацию и оптимизацию лидерных соединений** (lead identification and optimization) в процессе рационального дизайна лекарств. Более широко: применение методов информатики к решению любых химических задач — от предсказания физико-химических свойств молекул до разработки синтетических маршрутов и моделирования биологической активности.

**Классификация вычислительных методов по точности** строится вдоль нескольких осей:

**По физической основе (от точных к быстрым):**

1. **Квантовая химия (DFT, ab initio)** — наиболее точные методы, вычисляют волновые функции и электронную структуру. Применимы только к малым молекулам (до нескольких десятков атомов). Крайне трудоёмки: для каждого конформера требуется отдельный расчёт, что делает квантово-химический скрининг базы данных практически невозможным. Используются для одиночных молекул и переходных состояний.

2. **Силовые поля (молекулярная механика)** — атомы и связи описываются классической механикой (потенциалы Леннарда–Джонса, кулоновские взаимодействия, торсионные члены). Значительно быстрее квантовой химии, применяются в молекулярной динамике и конформационном поиске. Менее точны, особенно для ковалентно-химических событий.

3. **Дескрипторные/графовые методы** — молекулы представляются как объекты в химическом пространстве (векторы дескрипторов, молекулярные графы). Очень быстро, позволяют скринировать миллионы молекул. Качественный уровень точности, зависят от выбора дескрипторов. Основа QSAR/QSPR, виртуального скрининга, машинного обучения.

4. **ML-методы (машинное обучение, глубокое обучение)** — современные модели (AlphaFold, GNN, диффузионные модели) занимают промежуточное положение: они быстрее классических структурных методов и достигают высокой точности при предсказании структур белков. Скорость обработки: специализированное ПО типа NeuroClick — 10 000–30 000 молекул/мин.

**По типу методологии:**

- **Лиганд-базированные методы (быстрые)** — применяются когда структура мишени неизвестна, но есть набор активных молекул. Инструменты: молекулярное сходство (коэффициент Танимото), фармакофорное моделирование, QSAR. Молекулы сравниваются как лиганды, без явного учёта белка.

- **Структурно-базированные методы (медленные, точнее)** — требуют 3D-структуры белка (рентгеновская кристаллография, ЯМР, криоЭМ или гомологичное моделирование). Основной инструмент: **молекулярный докинг** (поиск позы и расчёт свободной энергии связывания ΔGbind). Дополнение: молекулярная динамика для оценки стабильности комплекса.

**Практический вывод:** в реальном пайплайне виртуального скрининга методы комбинируются — сначала быстрые фильтры (правило Липинского, ADMET-предсказания, фармакофорный скрининг), затем более точный докинг ограниченного числа кандидатов.

## ❓ Вопрос 2: Метрики (Python), используемые в хемоинформатике

> [!tip] Ключевые тезисы
> - Классификация: ROC-AUC, BEDROC, матрица ошибок (FP/FN)
> - Регрессия: RMSE, R², Q², коэффициент корреляции r
> - Библиотеки: sklearn, RDKit, PyRx
> - Стандарт: 5-кратная / 10-кратная кросс-валидация, split 80/20

### Развёрнутый ответ

В хемоинформатике модели решают две главные задачи: **классификацию** (активный/неактивный, проникает через ГЭБ / нет) и **регрессию** (предсказание IC₅₀, logBB, ΔGbind и т.д.). Для каждой — свой набор метрик.

**Метрики классификации:**

**ROC-AUC (Area Under the Receiver Operating Characteristic Curve)** — наиболее распространённая метрика. ROC-кривая строится в координатах чувствительность (Sensitivity/Recall) vs 1−специфичность. AUC = 1.0 → идеальная модель; AUC = 0.5 → случайное угадывание. В задачах ГЭБ-проницаемости лучшие модели (XGBoost + RDKit-дескрипторы) достигают AUC ≈ 0.959. В статье Шитьякова 2014: AutoDock AUC = 0.82, ADVina = 0.81. Реализация в Python: `sklearn.metrics.roc_auc_score`.

**BEDROC (Boltzmann-Enhanced Discrimination of Receiver Operating Characteristics)** — специализированная метрика для виртуального скрининга, измеряющая «раннее обогащение» (early recognition): насколько активные молекулы концентрируются в верхних позициях ранжированного списка. Важна потому, что в реальном скрининге физически тестируют только топ-N% списка.

**Матрица ошибок (Confusion Matrix):** TP, TN, FP, FN. На её основе рассчитываются:
- **Precision** = TP / (TP + FP)
- **Recall (Sensitivity)** = TP / (TP + FN)
- **F1-score** = 2 · Precision · Recall / (Precision + Recall)
- **Specificity** = TN / (TN + FP)
- **Matthews Correlation Coefficient (MCC)** — сбалансированная метрика для несбалансированных классов

Пример из практики: в статье 2014 года было обнаружено смещение по размеру молекулы — FN для соединений с < 20 тяжёлых атомов.

**Метрики регрессии:**

**RMSE (Root Mean Squared Error)** — корень из среднеквадратичной ошибки. Чувствителен к выбросам. Формула: `sqrt(mean((y_pred - y_true)²))`. Часто сообщают RMSEcv (по кросс-валидации) и RMSEtest (на тест-выборке). Реализация: `sklearn.metrics.mean_squared_error(squared=False)`.

**R² (коэффициент детерминации)** — доля дисперсии целевой переменной, объяснённая моделью. R² = 1 → идеальная модель; R² < 0 → модель хуже среднего. `sklearn.metrics.r2_score`.

**Q² (кросс-валидированный R²)** — показывает предсказательную силу модели на новых данных. Q² < R² указывает на переобучение. Стандарт для QSAR: Q² > 0.5 считается приемлемым.

**Корреляционный коэффициент (r или r²)** — Пирсон; для нелинейных зависимостей используется Спирмен.

**P-value** — статистическая значимость корреляции между дескриптором и биологической активностью.

**Техники валидации:**

**K-кратная кросс-валидация** (5-кратная — стандарт в хемоинформатике, 10-кратная — для малых датасетов): данные делятся на K частей, K−1 используются для обучения, 1 — для теста, процедура повторяется K раз. Разбиение обучение/тест: **80%/20%** — стандартная практика.

**Python-библиотеки:**

| Библиотека | Применение |
|-----------|-----------|
| `sklearn` | Метрики, SelectKBest, кросс-валидация, ML-алгоритмы (XGBoost, RandomForest) |
| `RDKit` | Расчёт молекулярных дескрипторов, генерация фингерпринтов (Morgan, MACCS, Avalon) |
| `numpy`, `scipy` | Линейная регрессия, статистика |
| `pandas` | Обработка датасетов |
| `PyRx` | Управление докингом AutoDock Vina, сбор результатов |
| `matplotlib`, `seaborn` | Визуализация ROC-кривых, регрессионных графиков |

## ❓ Вопрос 3: Преимущества и недостатки молекулярных скаффолдов

> [!tip] Ключевые тезисы
> - Скаффолд = ядро структуры (core), общее для серии соединений
> - Плюсы: репозиционирование лекарств, генерация библиотек, организация баз данных, снижение токсичности
> - Минусы: ограничение химического пространства, синтетическая труднодоступность, наследование плохого ADME

### Развёрнутый ответ

**Молекулярный скаффолд** (scaffold, «каркас») — это **ядровая структура** соединения или серии соединений, несущая основные фармакофорные элементы. Формально: кольцевая система + связывающие атомы (определение Мурко). Остальные части молекулы — заместители (R-группы), варьируемые для оптимизации активности и свойств.

**Преимущества:**

**1. Организация химических баз данных (Scaffold Tree).**
Иерархическая организация по ядрам позволяет навигировать в больших базах данных. Scaffold tree: удаление периферических колец → получение более простого ядра. Пример: аринг → серия молекул, сгруппированных по ядру, с которым связаны отдельные заместители. Эффективен для субструктурного поиска и анализа SAR.

**2. Репозиционирование лекарств (Drug Repurposing / Scaffold Searching).**
Поиск скаффолда среди одобренных FDA/EMA препаратов позволяет быстро находить кандидатов для новых показаний. Пример из курса: скаффолд трамипросата (пропансульфоновая кислота) → несколько клинически тестированных молекул как кандидаты при болезни Альцгеймера.

**3. Генерация комбинаторных библиотек.**
Донор скаффолда + список строительных блоков (R-групп) → виртуальная библиотека из сотен производных. Особенно эффективно в сочетании с click-химией (NeuroClick): ядро сохраняется, периферия комбинируется. Позволяет систематически исследовать SAR вокруг одного ядра.

**4. Улучшение физических свойств и специфичности.**
Скаффолды с высоким Fsp3 (fraction sp3 carbons) → более объёмные 3D-структуры → лучшее «вхождение» в активный сайт белка → повышенная селективность. Плоские (ароматические) скаффолды часто менее селективны.

**5. Снижение токсичности (токсикофорная замена).**
Известно, что некоторые субструктуры (токсикофоры) вносят нежелательный токсический потенциал. Замена скаффолда на изостерический аналог без токсикофора (scaffold hopping) позволяет сохранить активность при улучшении токсикологического профиля. Пример: виртуальные производные галоперидола с пониженным классом острой токсичности.

**Недостатки:**

**1. Ограничение химического пространства.**
Фиксированное ядро сужает исследуемое пространство: все производные имеют общую структуру, что снижает структурное разнообразие. Это особенно критично при поиске принципиально новых хемотипов (first-in-class candidates).

**2. Синтетическая труднодоступность производных.**
Генеративные модели и комбинаторные стратегии часто предлагают производные, которые крайне трудно или невозможно синтезировать. Частичное решение: виртуальная click-химия (реакции с известными реагентами → синтетически достижимые продукты), как в NeuroClick.

**3. Наследование нежелательных свойств.**
Производные могут унаследовать проблемные характеристики родительского скаффолда: высокий LogP → нарушение правил Липинского, плохая растворимость, нежелательные метаболические превращения (реактивные метаболиты). «Привязка» к ядру ограничивает свободу в оптимизации ADME.

**Ключевые концепции:**

**Scaffold Hopping** — переход к структурно отличному ядру при сохранении биологической активности. Используется для обхода патентной защиты и поиска молекул с улучшенными свойствами. Реализуется через замену фрагментов (fragment replacement) или биоизостерическую замену ядра.

**Scaffold Diversity** — максимизация структурного разнообразия ядер в скрининговой коллекции. Достигается через intramolecular reactions, cyclization strategies.

**Murcko Scaffold** — стандартный алгоритм выделения ядра: атомы кольцевых систем + линкеры между ними, без заместителей. Реализован в RDKit: `rdScaffoldNetwork.GetScaffoldForMol()`.

## 📄 Статья: Shityakov & Förster (2014) — BBB Choline Transporter

> [!summary] Краткий обзор для экзамена
> Авторы разработали in silico модель для скрининга молекул, способных использовать холиновый транспортёр ГЭБ (BBB-ChT) как вектор доставки в ЦНС. Поскольку BBB-ChT не кристаллизован, использовали гомологичную модель ChT1 (по шаблону PDB: 3DH4). Докинг 93 соединений (44 активных / 49 неактивных) с AutoDock и AutoDock Vina дал высокую корреляцию (r²=0.88) и AUC=0.82/0.81. Обнаружено смещение результатов для молекул с < 20 тяжёлых атомов (ложноотрицательные). Активный транспорт через BBB-ChT значительно усиливает проникновение катионных молекул в мозг по сравнению с пассивной диффузией.

> [!note] Подробный анализ
> Полный разбор статьи: [[Shityakov_Forster_2014_BBB_ChT]]

### Ключевые факты для экзамена
- **Мишень:** BBB-ChT (холиновый транспортёр ГЭБ) — не кристаллизован, используется гомологичная модель ChT1
- **Шаблон:** натрий/галактозный симпортер, PDB: 3DH4
- **Датасет:** 93 соединения (44 активных + 49 неактивных)
- **Метод:** молекулярный докинг (AutoDock + AutoDock Vina), PyRx
- **Главный результат:** r²=0.88, AUC=0.82/0.81, p<0.0001, SEE=0.775
- **Порог активности:** ΔGbind < −6.0 ккал/моль → BBB-ChT-активное
- **Ключевое открытие:** size-related bias — молекулы с <20 тяжёлых атомов получают ложноотрицательный результат докинга
- **Дескрипторы:** ΔGbind, ClogP, PSA, logBBpassive, logBBactive (с учётом pKiexp)
- **Вывод:** BBB-ChT можно использовать как вектор для ЦНС-доставки катионных лекарств; гомологичное моделирование + докинг = достаточно точная альтернатива QSAR
