# Теорминимум по хемоинформатике

> Единый конспект перед зачётом — без повторений, только суть.

---

## 1. Что такое хемоинформатика

**Хемоинформатика** — наука на стыке химии, математики и информатики для анализа химической информации.

Ключевая формула (Brown, 1998): **данные → информация → знание → лучшие решения быстрее**

Математически: молекула = точка в химическом пространстве, координаты = дескрипторы.

**Отличие от биоинформатики:**
- Хемоинформатика: малые органические молекулы, QSAR, докинг, виртуальный скрининг.
- Биоинформатика: геном, белки, AlphaFold, RNA-seq.
- Пересечение: белок-лигандные взаимодействия.

---

## 2. Пайплайн drug discovery

```
Базы данных (PubChem/ZINC/ChEMBL) →
Фильтрация (правило Липинского) →
Виртуальный скрининг (ligand-based + structure-based) →
Hit → Lead → Candidate → Клинические испытания
```

- **Хит** — молекула, показавшая активность в эксперименте
- **Лид** — лучший хит после оптимизации структуры
- **Кандидат** — лид, прошедший все критерии (ADME, токсичность, биодоступность)

---

## 3. Правило Липинского (Rule of Five)

Критерии пероральной биодоступности:
1. MW ≤ 500 Да
2. logP ≤ 5
3. H-доноры (HBD) ≤ 5
4. H-акцепторы (HBA) ≤ 10

Нарушение ≥2 критериев → плохая биодоступность. Также важны TPSA и структурные алерты.

---

## 4. Два подхода виртуального скрининга

**Ligand-based** (структура белка неизвестна):
- Поиск по сходству (similarity search, коэффициент Танимото)
- Фармакофорное моделирование
- QSAR/QSPR

**Structure-based** (известна 3D-структура белка):
- Молекулярный докинг

На практике воронка: миллионы молекул → ligand-based фильтры → докинг → единицы хитов.

---

## 5. Молекулярные дескрипторы

| Уровень | Примеры |
|---------|---------|
| 1D | MW, HBD, HBA |
| 2D | logP, TPSA, топологические индексы (Винера, Рандика), ECFP-фингерпринты |
| 3D | объём, SASA, Энтропия Вороного (VE) |

**Для ГЭБ критичны:** TPSA < 90 Å², logP ≤ 5, MW < 500 Да, HBD ≤ 3, logBB ≥ 0.3

**Уравнение Кларка:** `logBB = 0.152·logP − 0.0148·TPSA + 0.139`

**Энтропия Вороного:** `VE = −ΣPᵢ·ln(Pᵢ)`, где Pᵢ — доля площади i-го атома. Чем выше VE → тем асимметричнее молекула → тем выше аффинность (r²=0.94 на Mpro SARS-CoV-2).

---

## 6. Фингерпринты

**Фингерпринт** — битовый вектор наличия/отсутствия структурных фрагментов (0/1).

| Тип | Описание |
|-----|----------|
| **ECFP4** | Круговые, радиус 2 — самые популярные для ML |
| **MACCS keys** | 166 фиксированных фрагментов, интерпретируемы |
| **Daylight/RDKit** | Топологические пути в графе |

**Отличие от дескрипторов:**
- Дескрипторы → непрерывные числа → QSAR, регрессия
- Фингерпринты → биты → поиск сходства, классификация

**Сходство молекул — коэффициент Танимото:**
T(A,B) = |A ∩ B| / |A ∪ B|

T > 0.85 → молекулы похожи; T = 1.0 → идентичны.

---

## 7. Алгоритм Моргана / ECFP

1. Радиус 0: каждому атому → атомный номер (C→6, N→7, O→8)
2. Радиус R: суммируются значения соседей → кодирует окружение
3. Каждое число хэшируется → бит в векторе длиной 2048
4. ECFP4 = радиус 2, ECFP6 = радиус 3

---

## 8. QSAR / QSPR

**Концепция:** Y = f(D₁, D₂, ..., Dₙ), где Y — активность/свойство, Dᵢ — дескрипторы.

**Пайплайн:**
1. Данные из ChEMBL/PubChem (IC₅₀, logBB, Ki…)
2. Дескрипторы/фингерпринты
3. Обучение модели (80% train / 20% test + кросс-валидация)
4. Валидация: R², RMSE (регрессия), ROC-AUC (классификация)
5. Предсказание для новых молекул

**Домен применимости (Applicability Domain):** модель надёжна только для молекул, похожих на обучающую выборку. За пределами — экстраполяция ненадёжна.

**Переобучение:** Q² (кросс-валидация) должен быть близок к R². Большой разрыв → переобучение.

| Тип модели | Метод | Плюсы | Минусы |
|-----------|-------|-------|--------|
| Линейная | MLR, PLS | Интерпретируема | Не улавливает нелинейность |
| ML | Random Forest, XGBoost | Точность, нелинейность | Нужно много данных, «чёрный ящик» |
| 3D-QSAR | CoMFA, CoMSIA | Учитывает 3D-поля | Требует выравнивания структур |
| Фингерпринтовая | ECFP, MACCS | Быстро | Теряет геометрию |

## 9. Гематоэнцефалический барьер (ГЭБ)

- Защитный барьер кровь–мозг; не пропускает 98% лекарств
- Для ЦНС-препаратов проникновение **обязательно**, для периферических — **нежелательно**

**Ключевой параметр:** `logBB = log(C_мозг / C_кровь)`
- logBB ≥ 0.3 → хорошо проникает
- logBB < −1 → не проникает

**P-гликопротеин (P-gp)** — белок-«насос», активно выкачивает молекулы из мозга даже при хорошем TPSA. Проверяют через admetSAR или pkCSM.

**Инструменты предсказания ГЭБ:** FAF-Drugs4, admetSAR (BBB_P-Prob, AUC=0.952), pkCSM (LogPS > −2).

---

## 10. ADME/ADMET

| Буква | Что | Ключевые дескрипторы |
|-------|-----|---------------------|
| A — Absorption | Всасывание в ЖКТ | MW, logP, HBA/HBD, TPSA |
| D — Distribution | Распределение по тканям | logBB, P-gp |
| M — Metabolism | CYP450 ферменты печени | — |
| E — Excretion | Выведение через почки/желчь | — |
| T — Toxicity | LD50, hERG-канал, мутагенность | PAINS, Fsp3, PhI |

**Fsp3 > 0.4** → хороший 3D-профиль, меньше токсичности.

## 11. Фармакофор

**Фармакофор** — абстрактная 3D-модель требований: какие группы и где в пространстве нужны молекуле для связывания с белком. Это шаблон, а не конкретная молекула.

**Плюсы:** без структуры белка, быстрый скрининг, объясняет активность разных структур.
**Минусы:** не учитывает гибкость белка и сольватацию, требует набора активных молекул.

**Типы:** признаковые, нечёткие, лиганд-ориентированные, структурно-ориентированные, полевые (CoMFA/CoMSIA).

---

## 12. Молекулярный докинг

**Цель:** найти позу лиганда в сайте белка с минимальной свободной энергией Гиббса (ΔG, ккал/моль).

**ΔG < −8 ккал/моль** → хорошее связывание. Стандартная ошибка ≤ ±1.5 ккал/моль.

### Типы по гибкости:
1. Жёсткий рецептор / гибкий лиганд — стандарт
2. Оба гибкие — реалистичнее, вычислительно дорого
3. Оба жёсткие — геометрическое сопоставление

### 4 класса скоринговых функций:

| Класс | Суть | Программа |
|-------|------|-----------|
| **Силовые поля (FF)** | E_VdW + E_elec (Леннард-Джонс + Кулон) | AutoDock 4 |
| **Knowledge-based** | Статистика контактов из PDB | PMF-score, DrugScore |
| **Эмпирические** | Взвешенная сумма взаимодействий | AutoDock Vina, GlideScore |
| **Гибридные** | FF + эмпирика; ΔG = ΔH − TΔS | MM-PBSA/MM-GBSA |

### ПО для докинга:

| Программа | Алгоритм | Особенность |
|-----------|----------|-------------|
| AutoDock 4 | Генетический алгоритм (GA) | Эталон, bias для малых молекул |
| AutoDock Vina | BFGS (локальный поиск) | Быстро, многопоток |
| Glide (Schrödinger) | Иерархический | Высокая точность, дорого |
| GOLD | GA + учёт воды | Медленнее Vina |
| PatchDock | Геометрическое хэширование | Белок-белковый докинг |
| DiffDock (ML) | Диффузионная модель | 38% поз RMSD < 2Å |
| GNINA | MC + CNN | +5% vs AutoDock |

### Сайты связывания:
- **Активный (каталитический)** — место реакции (например, Asp25 в HIV-протеазе)
- **Ортостерический** — сайт природного лиганда
- **Аллостерический** — удалён от активного сайта, требует слепого докинга
- **Криптический** — появляется при мутациях/конформационных изменениях

### Стандартный протокол докинга (AutoDock Vina):
1. Скачать белок из PDB, проверить через PROCHECK (диаграмма Рамачандрана ≥90% в ядре)
2. Удалить воду, добавить полярные водороды, назначить заряды Гастайгера → формат **PDBQT**
3. Лиганд: SMILES → 3D (RDKit/OpenBabel), оптимизировать MMFF94, → PDBQT
4. Grid box: центр на активном сайте (20–30 Å), exhaustiveness = 8–32
5. Запуск → топ-9 поз с ΔG
6. **Валидация (обязательно):** редокинг ко-кристаллизованного лиганда, критерий RMSD < 2 Å

## 13. Алгоритмы оптимизации в докинге

### Генетический алгоритм (GA) — AutoDock 4:
- **Особь** = поза лиганда, **фитнес** = score
- Цикл: инициализация → оценка → отбор → скрещивание → мутация → новое поколение
- Стоп: достигнут max поколений ИЛИ score не улучшается

### Монте-Карло (MC):
- Случайный шаг → если лучше — принять; если хуже — принять с вероятностью exp(−ΔE/kT)
- **Критерий Метрополиса** позволяет выходить из локальных минимумов

### Имитация отжига (SA):
- Как MC, но температура T постепенно снижается: T = T₀·αˢᵗᵉᵖ (α ≈ 0.99)
- Высокая T → широкий поиск; низкая T → «замерзание» в минимуме

### Недостатки GA в докинге:
1. Жёсткий рецептор (induced-fit не учитывается)
2. Ограниченное пространство поиска (≤60 Å, blind docking неэффективен)
3. Size bias: малые молекулы недооцениваются, крупные — переоцениваются
4. Экспоненциальный рост конформаций с числом вращаемых связей
5. Риск застревания в локальном минимуме

## 14. Молекулярная динамика (МД)

**Суть:** решение F = ma для каждого атома на каждом шаге (~2 фс). Метод Верле/leap-frog:
```
F = −dU/dr  →  a = F/m  →  v(t+dt) = v(t) + a·dt  →  r(t+dt) = r(t) + v·dt
```

**Зачем МД после докинга?** Докинг — статичная «фотография», МД — «видео»: проверяет стабильность комплекса, учитывает гибкость белка и растворитель, даёт точную ΔG.

### Типы МД:

| Тип | Масштаб | Применение |
|-----|---------|------------|
| Классическая (ММ) | фс–мкс | Белок-лиганд, мембраны |
| Ab initio / QM/DFT | пс, ≤ сотни атомов | Хим. реакции, разрыв связей |
| QM/MM | нс, тысячи атомов | Ферментативные реакции |
| Coarse-grained | мкс–мс | Сворачивание белка, сборка мембран |

### ПО для МД:

| ПО | Силовые поля | Особенность |
|----|-------------|-------------|
| AMBER | ff14SB, GAFF, Lipid21 | MM-PBSA встроен; лучший для фармы |
| GROMACS | AMBER, CHARMM, GROMOS, OPLS | Быстрый, популярен в академии |
| NAMD | CHARMM, AMBER | Суперкомпьютеры, VMD |
| CHARMM | CHARMM36 | Нуклеиновые кислоты |
| Desmond (Schrödinger) | OPLS4 | 60–80× быстрее CPU |

Модели воды: TIP3P (быстрая), OPC (точная, с ff19SB).

### Стандартный протокол МД:
1. Подготовка белка (PROCHECK, водороды, заряды ff14SB) + лиганда (GAFF2, RESP-заряды)
2. Сольватация (TIP3P бокс, ионы Na⁺/Cl⁻ до 0.15 М)
3. Минимизация (500–5000 шагов steepest descent)
4. Нагрев NVT: 0 K → 300 K за 100–200 пс
5. Уравновешивание NPT: 1 атм, 300 K, 100 пс–1 нс
6. Продакшн: 50–500 нс (стандарт 100–200 нс)
7. Анализ: CPPTRAJ / MDAnalysis

### Ключевые параметры анализа МД:
- **RMSD** = √(1/N·Σ(rᵢ−r₀ᵢ)²) — стабильность; плато < 3 Å — норма; > 5 Å — что-то пошло не так. RMSD лиганда < 2 Å → докинг был верным.
- **RMSF** — гибкость по остаткам; коррелирует с B-фактором кристаллографии
- **Rg** (радиус вращения) — компактность белка
- **Водородные связи** — стабильность комплекса
- **MM-PBSA/MM-GBSA** — ΔG связывания (точнее докинга, медленнее)

## 15. Скаффолды

**Скаффолд** — общее химическое ядро для серии соединений.

| Задача | Суть |
|--------|------|
| **Scaffold Hopping** | Замена ядра при сохранении биоактивности |
| **Scaffold Diversity** | Максимально разные ядра в коллекции |
| **Scaffold Murcko** | Метод в RDKit для извлечения ядра |

**Scaffold Hopping** (типы по степени изменения):
1. Биоизостерическая замена (бензол → пиридин)
2. Топологически эквивалентные скаффолды
3. Фрагментный подход

**Плюсы:** обход патентной защиты, улучшение ADME, доступ к новым структурным классам.
**Минусы:** новое ядро может изменить 3D-геометрию и потерять активность.

## 16. Drug Repurposing (перепрофилирование)

**Суть:** поиск новых показаний для уже одобренных препаратов.
**Зачем:** традиционная разработка — 10–15 лет и >$1 млрд; одобренный препарат уже прошёл токсикологию.

**Пайплайн:**
1. Новая мишень определена (например, протеаза вируса)
2. Докинг всех одобренных препаратов из DrugBank (~10 000) к мишени
3. Отбор топ-10–50 по ΔG
4. ADME-проверка / similarity-поиск в ChEMBL
5. МД-валидация → in vitro/in vivo

**Пример:** лопинавир (ВИЧ) → докинг к 3CL-протеазе SARS-CoV-2 (ΔG ≈ −7..−8 ккал/моль) → клинические испытания. Гипотеза сгенерирована за недели вместо лет.

## 17. Белок-белковый докинг (PPI)

**Отличие от белок-лигандного:** лиганд ~70 атомов vs белок ~тысячи остатков; поиск в кармане 20 Å vs вся поверхность; минуты CPU vs часы на кластере.

| Программа | Алгоритм | Особенность |
|-----------|----------|-------------|
| ZDOCK, ClusPro | FFT (быстрое преобразование Фурье) | Быстро, без гибкости |
| HADDOCK | Эксперим. ограничения (NMR) + гибкость | Наиболее точный |
| RosettaDock | Монте-Карло | Перебор боковых цепей |

## 18. Сворачивание белка (Protein Folding)

**Иерархия структуры:**
1. Первичная — последовательность аминокислот
2. Вторичная — α-спирали и β-листы (водородные связи)
3. Третичная — 3D-конформация одной цепи (гидрофобный коллапс)
4. Четвертичная — комплекс нескольких цепей

**Парадокс Левинталя:** полный перебор конформаций белка из 100 аминокислот занял бы дольше возраста Вселенной. Решение — иерархическое сворачивание.

**Инструменты:**
- **AlphaFold2** — трансформер (Evoformer) + MSA; медианный RMSD ~0.56 Å; даёт 3D без PDB
- **RoseTTAFold** — трёхтрековая нейросеть, быстрее, чуть хуже точностью
- **Rosetta** — Monte Carlo протокол, RMSD ~2.6 Å
- **RetroFold** — логарифм. преобразование траектории разфолдинга → фолдинг за 100× меньше ресурсов (r²=0.99)

**Валидация 3D-структуры:** диаграмма Рамачандрана (PROCHECK) — ≥90% остатков в допустимых зонах.

## 21. ML/DL в хемоинформатике

**Представление молекулы для ML:**
- **Дескрипторы** — [MW, logP, TPSA, ...] → вектор чисел
- **Fingerprints (ECFP)** — [0,1,0,1,...] → битовый вектор
- **Граф** — атомы = узлы, связи = рёбра → для GNN

**Типы моделей:**

| Метод | Вход | Применение |
|-------|------|-----------|
| Random Forest | Дескрипторы/ECFP | logBB, токсичность |
| XGBoost | Дескрипторы/fingerprints | ГЭБ-классификация AUC=0.96 |
| ANN | Дескрипторы | Регрессия logBB |
| GNN (message passing) | Граф молекулы | EquiBind, предсказание позы |
| CNN (GNINA) | 3D-воксели | +5% vs AutoDock Vina |
| Диффузионная (DiffDock) | Граф + 3D | 38% поз RMSD < 2Å |
| VAE/GAN | SMILES/граф | De novo генерация молекул |

**ML vs классика:**
- ML: мгновенное предсказание для миллионов молекул, генерация структур
- Классика (GA, MC, силовые поля): не требует данных для обучения, физически интерпретируема
- Недостатки ML: нужно много данных, «чёрный ящик», стерические клэши при генерации

**Python-библиотеки:** RDKit, DeepChem, PyTorch Geometric, scikit-learn.

## 22. Теория графов в хемоинформатике

**Молекула → молекулярный граф:** атомы = вершины, связи = рёбра.

**Алгоритм Моргана → ECFP:** итеративно кодирует окружение каждого атома на заданный радиус.

**Применения:**
- Белок-белковые сети (PPI): белки = узлы, взаимодействия = рёбра; «хабы» — ключевые мишени
- Метаболические сети: метаболиты = узлы, ферменты = рёбра
- Граф де Брёйна — сборка NGS-данных (Velvet, SPAdes)

## 23. Синтетическая биология + хемоинформатика

**Пример: биосинтез артемизинина (противомалярийный)**

2. Ретросинтез (AiZynthFinder, ASKCOS) → дерево прекурсоров
3. Маппинг на ферменты (KEGG/Rhea)

- Квантовая химия (DFT) — для переходных состояний, только малые молекулы

## 24. SNP и мутации белков

**SNP** (Single Nucleotide Polymorphism) — замена одного нуклеотида.
- **Synonymous (синонимичная)** — белок не меняется
- **Non-synonymous (нссSNP):** missense (другая аминокислота) или nonsense (стоп-кодон)

**Предсказание патогенности:** SIFT, PolyPhen-2, PROVEAN, MutPred2, Snps&GO.

**Оценка влияния мутации на стабильность белка:**
- **FoldX** → ΔΔG: ΔΔG > 0 → дестабилизирует
- **Rosetta ddG** — Monte Carlo протокол; академический стандарт
- **In silico мутагенез** (PyMOL, AutoDock) → предсказать → затем экспериментальный мутагенез подтверждает

## 25. Секвенирование ДНК

| Поколение | Метод | Точность | Применение |
|-----------|-------|----------|-----------|
| I — Сэнгер | Капиллярный электрофорез | >99.9% | Проверка точечных мутаций |
| II — NGS/Illumina | Параллельное секвенирование | >99% | Полногеномное, RNA-seq |
| III — Nanopore | Пора в мембране | ~85–95% | Длинные риды (>10 000 нт), портативно |

**ПЦР** (полимеразная цепная реакция): Денатурация (94–98°C) → Отжиг праймеров (50–65°C) → Элонгация (72°C). За 30 циклов из 1 молекулы → ~10⁹ копий.

**Сайт-направленный мутагенез:**
1. Мутагенный праймер → ПЦР → плазмида с мутацией
2. Обработка DpnI (разрезает исходную метилированную ДНК)
3. Трансформация → секвенирование Сэнгера для подтверждения

## 26. Наноматериалы и супрамолекулярная химия

**Циклодекстрины** — конусообразные молекулы с гидрофобной полостью:
- α-CD (6 звеньев), β-CD (7), γ-CD (8)
- Образуют нековалентные комплексы «хозяин-гость» → повышают растворимость лекарств
- Докинг подбирает оптимальный тип под лиганд

**Наночастицы:**
- C60/C70 фуллерены — субстраты P-gp (ΔG = −54..−57 кДж/моль) → будут выкачаны
- SWCNT (нанотрубки) — НЕ субстрат P-gp (ΔG > 0) → перспективны для доставки лекарств при MDR

## 27. Белки-мишени для докинга

**Предпочтительные мишени:** белки с известной 3D-структурой + ко-кристаллизованным лигандом.

| Категория | Примеры |
|-----------|---------|
| Вирусные | HIV-1 протеаза (Asp25×2, C2-симметрия), SARS-CoV-2 Mpro |
| Бактериальные | β-дефенсин-1 (DEFB1) |
| ЦНС | Дофаминовые рецепторы D2/D3, серотониновые рецепторы |
| Онкологические | Bcl-2, амилоид-β (болезнь Альцгеймера) |

**Если нет кристаллической структуры:** гомологичная модель (SWISS-MODEL, Modeller) по родственному белку. Валидация: диаграмма Рамачандрана.

## Главные формулы для запоминания

```
Танимото:   T(A,B) = |A∩B| / |A∪B|
logBB:      logBB = 0.152·logP − 0.0148·TPSA + 0.139
ΔG докинг:  ΔG < −8 ккал/моль → хорошее связывание
RMSD:       RMSD = √(1/N · Σ(rᵢ − r₀ᵢ)²)  [норма < 3 Å в МД; < 2 Å при валидации докинга]
Критерий:   exp(−ΔE/kT)  [Метрополис — в MC и SA]
VE:         VE = −ΣPᵢ·ln(Pᵢ)  [Энтропия Вороного]
ECFP:       Алгоритм Моргана → хэш окружения атома → бит в векторе 2048
```
