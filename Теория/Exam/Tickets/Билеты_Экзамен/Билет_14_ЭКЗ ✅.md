## Вопрос 1: ML/DL методы в хемоинформатике

ML (машинное обучение) и DL (глубокое обучение) применяются для предсказания свойств молекул, виртуального скрининга и генерации новых структур.

**Как работает на практике — общий пайплайн:**
1. **Данные** — берут из ChEMBL/PubChem: молекулы + экспериментальные значения (IC₅₀, logBB, токсичность...)
2. **Представление молекулы** — выбор формы входа для модели (см. ниже)
3. **Обучение** — модель ищет связь между представлением и целевым свойством
4. **Валидация** — разбивка train/test (80/20), кросс-валидация, метрики ROC-AUC / RMSE
5. **Предсказание** — на новых молекулах без эксперимента

**Типы моделей и как они работают:**

| Метод                        | Вход                                        | Как работает                                                                               | ПО / библиотеки                          | Пример                               |
| ---------------------------- | ------------------------------------------- | ------------------------------------------------------------------------------------------ | ---------------------------------------- | ------------------------------------ |
| **Random Forest**            | Дескрипторы / ECFP-fingerprints             | Ансамбль решающих деревьев; усреднение → устойчив к шуму                                   | RDKit + scikit-learn                     | Предсказание logBB, токсичности      |
| **XGBoost**                  | Дескрипторы / fingerprints                  | Градиентный бустинг деревьев; последовательно исправляет ошибки                            | xgboost + scikit-learn                   | AUC = 0.96 для классификации ГЭБ     |
| **ANN (нейросеть)**          | Дескрипторы                                 | Слои: вход → скрытые → выход; обучается обратным распространением ошибки                   | PyTorch / TensorFlow / Keras             | Регрессия logBB, RMSE < 0.3          |
| **GNN (графовая нейросеть)** | Молекулярный граф (атомы=узлы, связи=рёбра) | Message passing: каждый атом агрегирует информацию от соседей → глобальный вектор молекулы | PyTorch Geometric, DGL-LifeSci, DeepChem | EquiBind — предсказание позы докинга |
| **Диффузионная модель**      | Граф + 3D-координаты                        | «Зашумляет» позу → учится обращать шум назад → генерирует реалистичные конформации         | DiffDock (MIT)                           | Success rate 38% (RMSD < 2 Å)        |
| **VAE / GAN**                | SMILES или граф                             | Кодирует молекулу в латентный вектор → декодирует новые структуры                          | GuacaMol, REINVENT (AstraZeneca)         | De novo генерация drug-like молекул  |
| **CNN (свёрточная нейросеть) — GNINA** | 3D-сетка вокселей белок-лиганд комплекса | CNN «видит» 3D-окружение кармана как изображение → предсказывает score позы; используется поверх AutoDock Vina: Vina генерирует позы → CNN переранжирует их | GNINA (открытый, github) | +5% success rate vs AutoDock Vina; гибрид классического поиска и DL-скоринга |

**Ключевые Python-библиотеки для ML в хемоинформатике:**
- **RDKit** — обработка молекул (SMILES → граф, дескрипторы, fingerprints)
- **DeepChem** — готовые ML-пайплайны для молекулярных данных
- **PyTorch Geometric** — GNN-архитектуры
- **scikit-learn** — классические ML (RF, SVM, логрегрессия)

## Вопрос 2: ML/DL vs классические методы

**Классические методы:**

- *Поисковые* — генетические алгоритмы (AutoDock Vina), Monte Carlo: быстрые, не требуют обучающих данных.
- *Физические* — форс-филды (MMFF94, AMBER): атомы как шары на пружинах. Квантовая химия (DFT): самая точная, но только для одной конформации, очень медленно.

**Преимущества ML/DL:**

1. Мгновенное предсказание свойств для миллионов молекул
2. Генерация новых структур (VAE, GAN, диффузия)
3. Интеграция разнородных данных

**Недостатки ML/DL:**

1. Нужны большие наборы данных; при малой выборке — ненадёжны
2. «Чёрный ящик» — сложно объяснить причину активности
3. Генерируют стерические клэши (атомы «слипаются»)
4. Плохо экстраполируют за пределы обучающих данных

## Вопрос 3: Конформационный поиск, симметрия, тесселляция Вороного

**Конформеры** — разные пространственные формы одной молекулы (отличаются углами вращения по одинарным связям).

Методы поиска:

1. *Систематический* — перебор углов с шагом 60°; точный, но дорогой (6ⁿ вариантов).
2. *Monte Carlo* — случайные изменения + критерий Метрополиса.
3. *Молекулярная динамика* — конформеры из снимков траектории.
4. *Генетические алгоритмы* — AutoDock Vina, до 32 вращаемых связей.

**Симметрия:** C2-симметрия ВИЧ-протеазы упрощает валидацию докинга; C3-симметрия — в DiffDock для параметризации поз.

**Тесселляция Вороного** — разбиение пространства: ячейка каждого атома содержит все точки, ближайшие именно к нему (не к соседним). Для молекулы из N атомов получается N многогранников (в 3D) или многоугольников (в 2D проекции).

Применения в хемоинформатике:
- **Дескриптор окружения атома** — площадь/объём ячейки отражает «упакованность»: маленькая ячейка = атом окружён плотно (гидрофобное ядро), большая = атом на поверхности
- **Анализ белок-лигандных контактов** — граница между ячейками белка и лиганда = контактная поверхность; позволяет точно определить, какие атомы взаимодействуют
- **QSAR-дескриптор** — объём ячейки атома коррелирует с его вкладом в липофильность и реакционную способность

**Энтропия Вороного (V_E)**:

$$V_E = -\sum_i P_i \cdot \log(P_i)$$

где $P_i = S_i / S_{total}$ — доля площади поверхности i-го атома от общей. 
- **V_E = 0** — абсолютно симметричная молекула (все ячейки одинаковы, например бензол)
- **V_E → max** — максимально несимметричная молекула (ячейки сильно различаются по размеру)

V_E используется в QSAR как топологический дескриптор формы молекулы — предсказывает биологическую активность без 3D-оптимизации.

## Статья: α-Циклодекстриновый димер для доставки допамина в мозг

**Зачем изучали:** при болезни Паркинсона DA и леводопа (L-DOPA) плохо проникают через ГЭБ и быстро разрушаются.

**Как делали:** 14 молекул (DA, L-DOPA, 12 Dopimid-производных) «состыковали» с α-CD-димером (полость 346 Å³) через AutoDock Vina; свойства рассчитали в PreADMET.

**Что получили:** лучшие кандидаты — Dopimid 9–11 (ΔG = −7.4…−7.6 ккал/моль, log BB > 0 — проникают через ГЭБ). Соединения 5–7 с этил-ацетатным остатком не вошли в полость (ΔG ≈ −4.3 ккал/моль). Гидрофобный дескриптор S_MHP (доля погружённой поверхности) коррелирует с константой связывания: r² = 0.87.

**Вывод:** α-CD-димер — рабочая наносистема доставки DA в ЦНС; структурные ветвления лиганда разрушают взаимодействие.

> [!important] Связь с вопросами билета
> Статья иллюстрирует докинг с конформационной оптимизацией (MMFF94), применение ADME-дескрипторов (log BB, log P) и геометрического дескриптора поверхности S_MHP — пример совместной работы поисковых алгоритмов и предсказания свойств в реальном проекте.
