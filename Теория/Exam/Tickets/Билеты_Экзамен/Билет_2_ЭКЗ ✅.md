## Вопрос 1: Два основных in silico метода в хемоинформатике

В хемоинформатике два подхода к виртуальному скринингу (отбору молекул на компьютере):

1. **Ligand-based** — структуру белка не знаем, работаем от активных молекул. Методы:
   - Фильтры по физ.-хим. методам (правило Липинского)
   - Поиск по similarity
   - **Фармакофорное моделирование** — определение ключевых пространственных и эл.-хим. характеристик → предсказание активности
   - **QSAR-моделирование**: молекула → хим. признаки (дескрипторы) → модель → предсказание активности. *Пример:* есть набор молекул с известным Toxicity → для каждой считают дескрипторы → строят мат./ML-модель, которая находит связь между признаками и активностью (Tox)

2. **Structure-based** — знаем структуру белка → молекулярный докинг. В VS последовательно: ligand-based → structure-based (воронка: миллионы → тысячи → единицы хитов).

## Вопрос 2: Почему ГЭБ критически важен для хемоинформатики?

ГЭБ (гематоэнцефалический барьер) — защитная стенка между кровью и мозгом из плотно сомкнутых клеток. Через него не проникает 98% потенциальных лекарств.

- Для препаратов против болезней Альцгеймера, Паркинсона, опухолей мозга — проникновение через ГЭБ **обязательно**.
- Для периферических препаратов — проникновение **нежелательно** (нейротоксичность).

Экспериментальная проверка (опыты на животных) дорогая и медленная — скринировать тысячи молекул нереально. Хемоинформатика решает это с помощью:

- Дескрипторов: TPSA (топологическая полярная поверхность) < 60–70 Å², LogP, молекулярный вес < 500 Да.
- Параметра **logBB** = log(концентрация в мозге / концентрация в крови) — **главный параметр проницаемости через ГЭБ**. Получают с помощью QSAR.
- Машинного обучения (Random Forest, нейросети) на больших датасетах.

**Пример QSAR — метод Кларка:** `log BB = 0.152 log P − 0.0148 TPSA + 0.139`

Важно учитывать P-гликопротеин (P-gp) — белок-«насос», который выкачивает молекулы обратно из мозга, даже если их TPSA в норме.

## Вопрос 3: Как хемоинформатика использует экспериментальные данные для обучения?

Модели в хемоинформатике невозможны без экспериментальных данных — они нужны для обучения и проверки.

**Два направления:**
![[Pasted image 20260624162541.png|459]]
1. **Индуктивное** (Данные → Знание): обучаем модель на экспериментальных IC₅₀, logBB, Ki → модель предсказывает свойства новых молекул. Пример: **QSAR** и метод **Кларка** (logBB).

2. **Дедуктивное** (Знание → Данные): знаем механизм → генерируем подходящие соединения. Пример: **NeuroClick** — знаем механизм клик-реакции → генерируем подходящие молекулы.

**Валидация:** разбивка 80/20, кросс-валидация, метрики R² и RMSE (среднеквадратичная ошибка). Цикл «данные → модель → предсказание → эксперимент → уточнение» — основа рационального дизайна лекарств.

## Статья: Дескрипторы полярной поверхности и проницаемость ГЭБ

**Зачем изучали:** Выяснить, какой дескриптор PSA (полярная поверхность молекулы) лучше всего предсказывает проникновение через ГЭБ.

**Как делали:** Сравнили три дескриптора — 2D-PSA, TPSA и 3D-PSA (рассчитанная квантово-механическим методом DFT) — на 19 препаратах; сопоставили с экспериментальными logBB через PLS-регрессию (метод частичных наименьших квадратов).

**Что получили:** 3D-PSA — лучший результат (R² = 0,92, RMSD = 0,29); TPSA чуть хуже (R² = 0,85), но рассчитывается быстрее без 3D-структуры. Порог: PSA < 60 Å² — молекула проникает через ГЭБ. Выбросы (risperidone, domperidone) — субстраты P-gp, которые выводятся активным транспортом.

**Вывод:** TPSA — оптимальный дескриптор для быстрого скрининга; 3D-PSA точнее, но дороже в расчёте; модели пассивной диффузии не работают при наличии P-gp.

> [!important] Связь с вопросами билета
> Статья — прямой пример ligand-based подхода (вопрос 1): дескрипторы PSA используются как фильтры. Одновременно показывает, как экспериментальные logBB служат основой для обучения регрессионной модели (вопрос 3) и почему ГЭБ — центральная тема хемоинформатики (вопрос 2).
