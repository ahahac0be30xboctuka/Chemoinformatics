## Вопрос 1: Алгоритмы молекулярного докинга

**Стохастические методы:**

**1. Генетический алгоритм (GA)** — вдохновлён эволюцией. Находит позу лиганда с min score.
- **Особь** = одна конкретная поза лиганда (координаты + углы вращения)
- **Ген** = набор параметров позы
- **Популяция** = набор из N поз одновременно
- **Фитнес** = значение скоринговой функции для этой позы (чем меньше, тем лучше)

Алгоритм — 1 поколение:
1. **Инициализация** — генерация случайных поз. Например, 100 поз
2. **Оценка** — score для каждой позы. Например: −8, −5, −9...
3. **Отбор** — лучшие позы выживают. Например, 50
4. **Скрещивание** — 2 позы объединяют часть генов → новая поза. Случайные пары из 50 лучших; как в природе: сильные особи размножаются чаще → лучшие позы имеют больший шанс. Пример:
   - Поза A (score −8): x=2.1, y=1.2, z=1.3, угол=90°, торсия=60°
   - Поза B (score −9): x=1.2, y=3.4, z=0.5, угол=45°, торсия=120°
   - Новая поза: x=1.2, y=1.2, z=0.5, угол=90°, торсия=120° ← выбор каждого параметра *случаен* (AutoDock)
5. **Мутация** — случайное изменение 1 параметра позы. Например, x=1.2 → x=1.4
6. **Оценка score**
7. **Отбор** — лучшие позы

Поколение 0 → лучшие позы → поколение 1 → ... → поколение N
Стоп: достигнуто max число поколений **ИЛИ** score между поколениями перестал улучшаться.

Результат: особь с лучшим score из всех поколений.

**2. Монте-Карло (MC)** — случайное изменение позы + решение о принятии изменения:
1. Score для нач. позы лиганда
2. Случайно сдвинуть 1 параметр → новая поза + новый score
3. Решение: а) новый score лучше → принять; б) новый score хуже → принять с вероятностью exp(−ΔE/kT) ← *критерий Метрополиса* (позволяет выйти из локального минимума)
4. Повторить
5. Конец: max число шагов / score перестал улучшаться

**3. Имитация отжига (SA, Simulated Annealing)** — расширение MC: та же идея, но температура T постепенно снижается по расписанию.

Зачем: при высокой T вероятность принять плохой ход велика → алгоритм широко исследует пространство поз (не застревает). При низкой T → вероятность падает → алгоритм «замерзает» вблизи найденного минимума. Аналогия: медленное охлаждение металла даёт кристаллическую структуру, быстрое — аморфную.

Алгоритм:
1. **Инициализация** — случайная поза лиганда; установить начальную температуру T₀ (высокую)
2. **Оценка** — score для текущей позы
3. **Случайный шаг** — сдвинуть 1 параметр → новая поза + новый score
4. **Решение** (критерий Метрополиса):
   - новый score лучше → принять всегда
   - новый score хуже → принять с вероятностью exp(−ΔE / kT)
   - при высокой T: exp(−ΔE/kT) близко к 1 → плохие ходы часто принимаются (широкий поиск)
   - при низкой T: exp(−ΔE/kT) → 0 → плохие ходы почти никогда не принимаются (локальная оптимизация)
5. **Снижение T** по расписанию: T = T₀ · α^шаг, например α = 0.99
6. **Повторить** с шага 3
7. **Стоп**: T опустилась до T_min **ИЛИ** score перестал улучшаться
Результат: поза, соответствующая глобальному (или близкому к нему) минимуму score.

**Систематические:** Glide (инкрементальное построение), DOCK (graph-matching — сопоставление геометрии лиганда и сайта).

**Методы машинного обучения (ML)**: DiffDock (генеративная диффузионная модель), EquiBind/TankBind (регрессионные, предсказывают координаты напрямую), GNINA (гибрид: MC + свёрточная нейросеть (CNN) для переоценки поз, +5% vs. AutoDock)

## Вопрос 2: Принципы машинного обучения в хемоинформатике

**Базовая идея:** модель ищет математическую зависимость между представлением молекулы и целевым свойством (активность, токсичность, ΔG...).

**1. Представление молекулы (вход модели)**
Модель не понимает «молекулу» — ей нужен числовой вектор:
- **Дескрипторы** — числовые свойства: MW, logP, TPSA, ... → вектор [500, 3.2, 120, ...]
- **Fingerprints (ECFP)** — битовый вектор наличия структурных фрагментов: [0,1,0,1,1,0,...]
- **Граф** — атомы = узлы, связи = рёбра; для GNN

**2. Типы задач**
- **Классификация** — активная молекула или нет? (выход: 0/1). Метрики: ROC-AUC, F₁, BEDROC
- **Регрессия** — предсказать числовое значение (IC₅₀, logBB, ΔG). Метрики: RMSE, R²
- **Генерация** — создать новые молекулы с нужными свойствами (VAE, GAN, диффузия)

**3. Цикл обучения**
```
Данные (ChEMBL/PDB) → представление молекул → разбивка train/test (80/20)
→ модель минимизирует ошибку на train → валидация на test → предсказание
```

**4. Ключевые принципы**
- **Переобучение (overfitting):** модель запомнила обучающие данные, но плохо работает на новых. Контроль: Q² на кросс-валидации должен быть близок к R²
- **Кросс-валидация:** данные делятся на k частей (обычно k=5); модель обучается на k−1, тестируется на 1; повторяется k раз → усреднение метрик
- **Домен применимости (applicability domain):** модель надёжна только для молекул, похожих на обучающую выборку; за пределами — экстраполяция ненадёжна
- **Несбалансированность классов:** активных молекул обычно <<< неактивных → нужна BEDROC/EF вместо accuracy

**5. Применение в докинге (примеры)**
- **GNINA** — CNN заменяет эмпирическую скоринговую функцию: классический поиск (Vina) + нейросетевое переранжирование поз
- **DiffDock** — генеративная диффузионная модель: «зашумляет» позу → учится обращать шум → 38% поз с RMSD < 2 Å
- **EquiBind/TankBind** — регрессия: сеть напрямую предсказывает координаты позы (быстро, но даёт клэши)

Главный недостаток ML в докинге: нет физической скоринговой функции — confidence-score ≠ аффинность связывания.

## Вопрос 3: Недостатки генетических алгоритмов в молекулярном докинге

1. **Жёсткий рецептор:** белок зафиксирован, induced-fit (подстройка белка под лиганд) не учитывается.
2. **Ограниченное пространство поиска:** нужна кубическая сетка (≤ 60 Å). Blind docking (поиск по всему белку) неэффективен.
3. **Size-related bias:** малые молекулы недооцениваются, крупные переоцениваются из-за размера.
4. **Экспоненциальный рост:** каждая вращаемая связь умножает число конформаций — гарантии нахождения глобального минимума нет.
5. **Риск застревания:** при малом числе итераций (GA-runs) алгоритм застревает в локальном минимуме.
6. **Нет параметров:** если для атомов лиганда нет параметров в силовом поле — докинг невозможен.

## Статья: Scaffold Searching in Alzheimer's Disease (Shityakov et al., 2021)

**Зачем изучали:** Найти одобренные препараты (drug repurposing — повторное применение лекарств) с активностью против β-амилоидного пептида (Aβ) при болезни Альцгеймера.

**Как делали:** В DrugCentral (4 642 препарата) провели скаффолд-поиск по структуре трамипросата (TRA — провалившегося ингибитора Aβ); 13 хитов проверили докингом (AutoDock 4) к Aβ-фибриллам (PDB: 2NAO), стабильность подтвердили 100 нс молекулярной динамикой (MM-GBSA).

**Что получили:** Два лидера — менадион бисульфит (ΔG = −7.11 ккал/моль) и камфотамид (ΔG = −6.82 ккал/моль) — превзошли TRA (ΔG = −6.19 ккал/моль) по аффинности и проницаемости через ГЭБ (гематоэнцефалический барьер; logBB камфотамида = −0.37 vs. −0.78 у TRA); MM-GBSA: менадион ΔG_GB = −19.71, TRA = −10.1 ккал/моль.

**Вывод:** Скаффолд-поиск → докинг → МД — рабочий in silico пайплайн для раннего отбора кандидатов.

> [!important] Связь с вопросами билета
> Статья напрямую демонстрирует применение AutoDock (генетический алгоритм) для виртуального скрининга и использует MM-GBSA как более точную альтернативу эмпирической скоринговой функции — именно то, о чём говорят вопросы 1–3.
