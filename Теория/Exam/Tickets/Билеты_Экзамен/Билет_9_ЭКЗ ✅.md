## Вопрос 1: Вычислительная синтетическая биология и хемоинформатика

**Синтетическая биология (SynBio)** — проектирование и конструирование новых биологических частей, устройств и систем, а также перепроектирование существующих природных биологических систем для полезных целей.
Применяет принцип Design → Build → Test → Learn к живым системам. 

**Хемоинформатика** — представление молекул как математических объектов (графов, векторов) и предсказание их свойств компьютерными методами (QSAR, докинг, виртуальный скрининг).

Пример пересечения - *биосинтез артемизинина*(противомалярийный препарат):

1. **Генеративная химия** (хемоинформатика): 
   Модель (VAE / диффузионная) предлагает молекулы с нужными свойствами. На выходе — SMILES целевого соединения.
2. **Ретросинтез** (хемоинформатика):
   ASKCOS / AiZynthFinder рекурсивно разбивают SMILES на прекурсоры:
```
Артемизинин ↑ Артемизининовая кислота ↑ Amorphadiene ↑ FPP (уже есть в клетке)
```
3. **Маппинг на ферменты** (стык)
   Каждый химический шаг из дерева — это по сути реакция. Для каждой реакции ищем фермент, который её катализирует, в базах KEGG/Rhea. Нашли: ADS превращает FPP в amorphadiene, CYP71AV1 превращает amorphadiene в кислоту. Оба фермента существуют в природе — в полыни.
   
4. **Реализация** (SynBio):
Теперь нужно «пересадить» эти ферменты в дрожжи. Берём гены ADS и CYP71AV1, синтезируем их искусственно (адаптируя под дрожжевую клетку), встраиваем в геном _S. cerevisiae_. Дополнительно усиливаем собственный путь дрожжей, который производит FPP — чтобы сырья для реакций было больше.

**Результат:** >100 мг/л артемизининовой кислоты (Keasling lab, 2006) → промышленное производство (Sanofi).

## Вопрос 2: Теория графов в синтетической биологии и хемоинформатике

**Граф G = (V, E)** — множество вершин V и рёбер E. Это универсальный способ кодировать структурную информацию.

**В хемоинформатике:**
Молекула → **молекулярный граф**: атомы = вершины, связи = рёбра. Это позволяет применять математический аппарат теории графов: считать кратчайшие пути, кодировать окружения атомов, сравнивать структуры.

**Зачем нужна теория графов?** Компьютер не понимает «молекулу» — ему нужны числа. Граф даёт однозначный способ перевести структуру в числовое представление (дескрипторы, фингерпринты), пригодное для ML + можно сравнивать молекулы через фингерпринты и коэффициент Танимото:

**Схема:** Молекула → граф → фингерпринт → сравнение.

**Алгоритм Моргана** позволяет закодировать не только тип атома, но и его **химическое окружение** на несколько связей вокруг (позволяет отличить, например, атом C в бензольном кольце от C в алифатической цепи)

*Как работает (пошагово):*
1. Радиус 0: каждому атому присваивается его атомный номер (C→6, N→7, O→8)
2. Радиус 1: суммируются значения соседей → кодирует ближайшее окружение
3. Радиус R: повторяем → ECFP4 (радиус 2), ECFP6 (радиус 3)
4. Каждое число хэшируется → бит в векторе длиной 2048

*Результат:* **ECFP-фингерпринт** — битовый вектор, где каждый бит означает наличие определённого структурного фрагмента. 

Сравнение молекул = **коэффициент Танимото:**
$$
T(A,B) = |A ∩ B| / |A ∪ B|
$$
- |A ∩ B| — биты = 1 у обеих молекул
- |A ∪ B| — биты = 1 хотя бы у одной
- T = 1.0 → идентичны, T > 0.85 → похожи, T = 0.0 → ничего общего

**Отличие от простого фингерпринта:** ECFP учитывает **локальное химическое окружение** каждого атома, а не просто наличие фрагмента. Поэтому он точнее кодирует структуру.

**В синтетической биологии:**
- **Белок-белковые сети (PPI)**: белки = вершины, взаимодействия = рёбра; «хабы» — ключевые мишени
- **Метаболические сети**: метаболиты = вершины, ферменты = рёбра
- **Геномная сборка**: граф **де Брёйна** — основа сборки NGS-данных (Velvet, SPAdes)

## Вопрос 3: QSAR и QSPR — концепция и типы

**QSAR** (Quantitative Structure–Activity Relationship) — количественная **концепция**, связывающая структуру молекулы с её биологической активностью (IC₅₀, Ki, LD₅₀).

**QSPR** (Quantitative Structure–Property Relationship) — то же, но целевое значение — физико-химическое свойство: logP, logBB, TPSA, растворимость.

Общая формула: 
$$
Y = f(D₁, D₂, ..., Dₙ), 
$$
где Y — активность/свойство, Dᵢ — дескрипторы молекулы.

**Как строится QSAR/QSPR модель (пайплайн):**
1. **Данные** — набор молекул с известными экспериментальными Y (например, IC₅₀ из ChEMBL)
2. **Дескрипторы** — каждая молекула кодируется числовым вектором: MW, logP, TPSA, ECFP-fingerprints...
3. **Обучение модели** — алгоритм ищет зависимость Y = f(дескрипторы)
4. **Валидация** — разбивка train/test, кросс-валидация; метрики: R², RMSE (регрессия), ROC-AUC (классификация)
5. **Предсказание** — для новой молекулы считаем дескрипторы → подставляем в модель → получаем Y без эксперимента

**Домен применимости (Applicability Domain):** модель надёжна только для молекул, похожих на обучающую выборку. За его пределами — экстраполяция, результаты ненадёжны.

| Тип                 | Метод                                  | Плюсы                                     | Минусы                                    |
| ------------------- | -------------------------------------- | ----------------------------------------- | ----------------------------------------- |
| **Линейные**        | MLR, PLS                               | Интерпретируемы, работают на малых данных | Не улавливают нелинейность                |
| **ML-методы**       | Random Forest, XGBoost, SVM, нейросети | Высокая точность, нелинейность            | Нужно много данных, «чёрный ящик»         |
| **3D-QSAR**         | CoMFA, CoMSIA                          | Учитывают 3D-форму и поля молекулы        | Требуют корректного выравнивания структур |
| **Фингерпринтовые** | ECFP, MACCS                            | Быстро, легко, хорошо для similarity      | Теряют информацию о геометрии             |

**Конкретные примеры QSPR:**
- Метод Кларка: `logBB = 0.152·logP − 0.0148·TPSA + 0.139` — предсказывает проникновение через ГЭБ
- logP по методу Wildman–Crippen — рассчитывается из атомных вкладов
- XGBoost для классификации ГЭБ: ROC-AUC = 0.96 (NeuroClick)

**Зачем QSAR/QSPR**
Это единственный способ предсказать свойство молекулы *без синтеза и эксперимента* — именно поэтому они лежат в основе любого виртуального скрининга и пайплайна drug discovery.

## Статья: NeuroClick (Isakova et al., 2024)

**Зачем изучали:** разработать программу для автоматической генерации библиотек лекарственно-подобных молекул методом in silico click-реакции с фильтром проникновения через ГЭБ.

**Как делали:** Python/RDKit программа с GUI имитирует азид-алкиновое циклоприсоединение (реагенты в формате SMILES) и фильтрует продукты по правилу Липинского и уравнениям QSPR Clark/Rishton (порог logBB ≥ 0.3).

**Что получили:** из 143 075 азидов + 1 алкин сгенерировано 284 770 триазолов за ~17 мин (~9 000 молекул/мин); после двойной фильтрации (Липинский + Clark) осталось **1 056 кандидатов**, 95% из которых не являются субстратами Pgp (белка множественной лекарственной устойчивости).

**Вывод:** NeuroClick объединяет click-химию и хемоинформатику в одном инструменте, превосходя устаревший AutoClickChem по функциональности.

> [!important] Связь с вопросами билета
> Статья — прямая иллюстрация вопросов 1–3: NeuroClick пересекает синтетическую биологию (click-реакция) с хемоинформатикой (SMILES/граф, фильтрация QSPR), а встроенные уравнения Clark и Rishton — конкретные QSPR-модели для предсказания ГЭБ-проникаемости.
